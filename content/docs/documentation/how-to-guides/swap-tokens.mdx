## Overview

The Masumi Payment Service includes a built-in integration with [SundaeSwap](https://sundaeswap.finance/), a Cardano DEX. You can use it to swap ADA for Cardano Native Tokens (CNTs) or CNTs back to ADA â€” directly from your node's API, without leaving the Masumi ecosystem.

This is useful for keeping your wallets funded with the right tokens (e.g., converting ADA revenue to USDM or USDCx for purchases).

### Supported tokens

| Token  | Policy ID | Asset Name (hex) |
|--------|-----------|-----------------|
| ADA    | *(native)* | `""` (empty string) |
| USDM   | `c48cbb3d5e57ed56e276bc45f99ab39abe94e6cd7ac39fb402da47ad` | `0014df105553444d` |
| USDCx  | `1f3aec8bfe7ea4fe14c5f121e2a92e301afe414147860d557cac7e34` | `5553444378` |

## Typical Swap Workflow

<Steps>
<Step>
**Get a price estimate**

Before executing a swap, fetch a live estimate from the pool to check the current rate and fee.

```bash
curl -X GET "http://localhost:3001/api/v1/swap/estimate/" \
  -H "token: YOUR_ADMIN_KEY" \
  -G \
  --data-urlencode "fromPolicyId=" \
  --data-urlencode "fromAssetName=" \
  --data-urlencode "toPolicyId=c48cbb3d5e57ed56e276bc45f99ab39abe94e6cd7ac39fb402da47ad" \
  --data-urlencode "toAssetName=0014df105553444d" \
  --data-urlencode "poolId=64f35d26b237ad58e099041bc14c687ea7fdc58969d7d5b66e2540ef"
```

Response:

```json
{
  "rate": 2.45,
  "fee": 0.003,
  "fromDecimals": 6,
  "toDecimals": 6
}
```

`rate` is the estimated output per 1 unit of input, after the pool fee.
</Step>

<Step>
**Execute the swap**

```bash
curl -X POST "http://localhost:3001/api/v1/swap/" \
  -H "token: YOUR_ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "walletVkey": "your_wallet_vkey",
    "amount": 10,
    "fromToken": {
      "policyId": "",
      "assetName": "",
      "name": "ADA"
    },
    "toToken": {
      "policyId": "c48cbb3d5e57ed56e276bc45f99ab39abe94e6cd7ac39fb402da47ad",
      "assetName": "0014df105553444d",
      "name": "USDM"
    },
    "poolId": "64f35d26b237ad58e099041bc14c687ea7fdc58969d7d5b66e2540ef",
    "slippage": 0.03
  }'
```

| Field | Description |
|-------|-------------|
| `walletVkey` | Verification key of the wallet funding the swap |
| `amount` | Amount to swap (in ADA or token units) |
| `fromToken` / `toToken` | Source and destination token. Use empty strings for ADA's `policyId` and `assetName` |
| `poolId` | SundaeSwap liquidity pool identifier |
| `slippage` | Maximum acceptable price deviation (0â€“1, default `0.03` = 3%) |

Response:

```json
{
  "txHash": "abc123...",
  "walletAddress": "addr1..."
}
```
</Step>

<Step>
**Poll for confirmation**

Swap orders are placed at the SundaeSwap script address and filled asynchronously by the DEX. Poll until `status` is `confirmed` and `swapStatus` reaches `Completed`.

```bash
curl -X GET "http://localhost:3001/api/v1/swap/confirm/" \
  -H "token: YOUR_ADMIN_KEY" \
  -G \
  --data-urlencode "txHash=abc123..." \
  --data-urlencode "walletVkey=your_wallet_vkey"
```

Response:

```json
{
  "status": "confirmed",
  "swapStatus": "OrderConfirmed",
  "confirmations": 5
}
```

| `swapStatus` value | Meaning |
|--------------------|---------|
| `OrderPending` | Order transaction submitted, not yet on-chain |
| `OrderConfirmed` | Order is at the DEX script address, waiting to be filled |
| `Completed` | DEX has filled the order â€” tokens are in your wallet |
| `CancelPending` | Cancel transaction submitted |
| `CancelConfirmed` | Order successfully cancelled |
</Step>
</Steps>

## Cancelling a Pending Order

If your order is in `OrderConfirmed` state (sitting at the DEX script), you can cancel it to recover your funds:

```bash
curl -X POST "http://localhost:3001/api/v1/swap/cancel/" \
  -H "token: YOUR_ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "walletVkey": "your_wallet_vkey",
    "swapTransactionId": "clx1abc..."
  }'
```

Response includes a `cancelTxHash` you can use to track the cancellation on-chain.

## Listing Past Swap Transactions

```bash
curl -X GET "http://localhost:3001/api/v1/swap/transactions/" \
  -H "token: YOUR_ADMIN_KEY" \
  -G \
  --data-urlencode "walletVkey=your_wallet_vkey" \
  --data-urlencode "limit=10"
```

Results are ordered most-recent-first and support cursor-based pagination via `cursorId`.

## Handling Timeouts

Occasionally a swap order transaction may time out before it's confirmed on-chain. Use the `acknowledge-timeout` endpoint to check on-chain state and recover to the correct status:

```bash
curl -X POST "http://localhost:3001/api/v1/swap/acknowledge-timeout/" \
  -H "token: YOUR_ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "walletVkey": "your_wallet_vkey",
    "swapTransactionId": "clx1abc..."
  }'
```

The service checks the chain and recovers to one of:
- `OrderConfirmed` â€” order UTXO still exists, you can retry or cancel
- `Completed` â€” DEX filled the order despite the timeout
- Timeout state unchanged â€” order transaction never made it on-chain
