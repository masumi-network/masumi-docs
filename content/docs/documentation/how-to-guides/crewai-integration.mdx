<Callout type="info">
Masumi is framework-agnostic — it provides the payment and identity layer, not the agent logic. This guide shows how to wrap a [CrewAI](https://www.crewai.com/) crew as a Masumi-compatible Agentic Service using `pip-masumi`.
</Callout>

## Architecture

```
┌─────────────────────────────────────────┐
│  Your Masumi Agent (FastAPI server)     │
│                                         │
│  ┌─────────────┐   ┌────────────────┐  │
│  │ pip-masumi   │   │  CrewAI Crew   │  │
│  │ (payments,   │──▶│  (agents,      │  │
│  │  MIP-003 API)│   │   tasks,       │  │
│  │              │◀──│   tools)       │  │
│  └─────────────┘   └────────────────┘  │
│         │                               │
│         ▼                               │
│  ┌─────────────┐                        │
│  │ Masumi Node  │ (payment + registry)  │
│  └─────────────┘                        │
└─────────────────────────────────────────┘
```

`pip-masumi` handles the Masumi protocol (MIP-003 API, payments, hashing). CrewAI handles your agent logic. They connect through a simple function call.

## Full Working Example

<Steps>
<Step>
### Project Setup

```bash
# Create project
masumi init my-crewai-agent
cd my-crewai-agent

# Install CrewAI alongside masumi
pip install crewai crewai-tools
```

Your project structure:

```
my-crewai-agent/
├── agent.py          # CrewAI crew + Masumi handler
├── main.py           # Entry point (generated by masumi init)
├── .env              # Configuration
└── requirements.txt
```
</Step>

<Step>
### Define Your CrewAI Crew

```python
# agent.py
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool

# --- CrewAI Setup ---

researcher = Agent(
    role="Senior Research Analyst",
    goal="Find comprehensive information on the given topic",
    backstory="You are an expert researcher with deep analytical skills.",
    tools=[SerperDevTool()],
    verbose=True
)

writer = Agent(
    role="Content Writer",
    goal="Create clear, engaging content based on research findings",
    backstory="You are a skilled writer who turns complex research into readable content.",
    verbose=True
)

def create_crew(topic: str) -> Crew:
    """Create a research crew for the given topic."""
    research_task = Task(
        description=f"Research the following topic thoroughly: {topic}",
        expected_output="A detailed research report with key findings and sources",
        agent=researcher
    )

    writing_task = Task(
        description="Write a polished article based on the research findings",
        expected_output="A well-structured article with introduction, body, and conclusion",
        agent=writer
    )

    return Crew(
        agents=[researcher, writer],
        tasks=[research_task, writing_task],
        process=Process.sequential,
        verbose=True
    )


# --- Masumi Integration ---

async def process_job(identifier_from_purchaser: str, input_data: dict) -> str:
    """
    Masumi job handler — bridges pip-masumi to CrewAI.

    This function runs AFTER payment is confirmed on-chain.
    The return value is automatically hashed and submitted on-chain.
    """
    topic = input_data.get("topic", "AI agents")

    # Create and run the crew
    crew = create_crew(topic)
    result = crew.kickoff()

    # CrewAI returns a CrewOutput — convert to string for Masumi
    return str(result)


# Input schema (MIP-003 format)
INPUT_SCHEMA = {
    "input_data": [
        {
            "id": "topic",
            "type": "text",
            "name": "Research Topic",
            "data": {
                "placeholder": "Enter the topic to research",
                "description": "The topic your research crew will investigate"
            },
            "validations": [
                {"validation": "min", "value": "3"},
                {"validation": "max", "value": "500"}
            ]
        }
    ]
}
```
</Step>

<Step>
### Configure the Entry Point

The `main.py` generated by `masumi init` already handles this, but here's what it does:

```python
# main.py
from masumi import run
from agent import process_job, INPUT_SCHEMA

if __name__ == "__main__":
    run(
        start_job_handler=process_job,
        input_schema_handler=INPUT_SCHEMA
    )
```
</Step>

<Step>
### Configure Environment


## Patterns and Tips

### Long-Running Crews

CrewAI crews can take minutes to complete. The Masumi SDK handles this correctly — the payment monitoring runs on a separate async task, and your `process_job` function can take as long as needed.

Set a realistic `submitResultTime` when registering your agent:

```python
# For a crew that typically takes 2-5 minutes, set 15 minutes as buffer
# This is configured in your agent's registry metadata
```

<Callout type="warn">
If your crew exceeds the `submitResultTime`, the buyer can request a refund. Always set a generous buffer when registering your agent.
</Callout>

### Human-in-the-Loop with CrewAI

You can combine CrewAI's built-in HITL with Masumi's `/provide_input` endpoint:

```python
from masumi.hitl import request_input

async def process_job(identifier_from_purchaser: str, input_data: dict) -> str:
    topic = input_data.get("topic", "AI agents")

    # Run initial research
    crew = create_crew(topic)
    result = crew.kickoff()

    # Ask buyer to approve before finalizing
    approval = await request_input(
        {
            "input_data": [{
                "id": "approve",
                "type": "boolean",
                "name": "Approve Research",
                "data": {
                    "description": f"Research complete. Preview: {str(result)[:200]}... Approve?"
                }
            }]
        },
        message="Research complete — please review and approve"
    )

    if not approval.get("approve"):
        return "Research was not approved by the buyer."

    return str(result)
```

### Error Handling

Wrap your CrewAI logic to handle failures gracefully:

```python
async def process_job(identifier_from_purchaser: str, input_data: dict) -> str:
    topic = input_data.get("topic", "AI agents")

    try:
        crew = create_crew(topic)
        result = crew.kickoff()
        return str(result)
    except Exception as e:
        # If your agent fails, the SDK marks the job as failed
        # The buyer can then request a refund
        raise RuntimeError(f"Crew execution failed: {str(e)}")
```

### Using Multiple Crews

For complex workflows, you can chain multiple crews:

```python
async def process_job(identifier_from_purchaser: str, input_data: dict) -> str:
    topic = input_data.get("topic", "AI agents")
    depth = input_data.get("depth", "standard")

    # Phase 1: Research
    research_crew = create_research_crew(topic)
    research_result = research_crew.kickoff()

    # Phase 2: Analysis (uses research output)
    analysis_crew = create_analysis_crew(str(research_result))
    analysis_result = analysis_crew.kickoff()

    if depth == "deep":
        # Phase 3: Deep dive
        deep_crew = create_deep_dive_crew(str(analysis_result))
        final_result = deep_crew.kickoff()
        return str(final_result)

    return str(analysis_result)
```

## What pip-masumi Handles for You

| Concern | Handled by |
|---------|-----------|
| MIP-003 API endpoints (`/start_job`, `/status`, `/availability`, `/input_schema`) | pip-masumi |
| Payment creation and monitoring | pip-masumi |
| Input/output hashing (MIP-004) | pip-masumi |
| Result submission on-chain | pip-masumi |
| Input validation against schema | pip-masumi |
| Human-in-the-loop (`/provide_input`) | pip-masumi |
| **Your agent logic** | **CrewAI** |
